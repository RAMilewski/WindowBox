Today, with the help of ...... I installed an LG Ultrawide monitor in the window at Spark.  It's mounted in portrait mode to fit the window better, and driven by a RaspberryPi Zero 2W.

Since the original monitor stand only allowed for landscape orientation, I desinged a custom VESA mount in OpenSCAD that attaches to the window bars.

The software is a python program with a playlist that rotates graphic images on a predetermined schedule.   The ultrawide monitor proved to be a bit of a challenge because it has an aspect ratio of 2.35:1 and LG did not properly implement EDID, so it assumes 16:9 images an stretches them to 2.35:1.  The program accepts 2.35:1 images but compresses them to 16:9 before handing them to the monitor which re-expands them to 2.35:1.  

The software contains switches so if anyone wants to use the software to drive normal monitors (in either portrait or landscape orientation), it's simple to change.

The project is on GitHub at https://https://github.com/RAMilewski/WindowBox.  I know very little Python so I played project manager and wrote an initial spec and handed it to Anthropics Claude to do the coding.  The initial spec is in the file Claude.md in the repository, however Claude and I went through about a dozen iteration making changes that aren't in that initial spec.

Installation and operation instructions are in RasPi.md, and there is a PDF version of that doc as well in RasPi.pdf.

If you would like to add a graphic to the rotation it needs to be in portrait orientation and 2530x1080 pixels in size.  Submit it as a pull request to GitHub.  You can either make edits to playlist.txt and/or priority.txt (see the docs), or just put a remark in the PR saying when you would like the graphic to play.

I've included the SCAD file for the mount in the repository just for completeness, but I doubt it will be of use to anyone else as our mounting requirements here are a bit strange.
